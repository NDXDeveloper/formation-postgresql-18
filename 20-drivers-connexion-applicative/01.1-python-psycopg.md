# 20.1.1. Python : psycopg3 (psycopg2 legacy)

## Introduction

Python est l'un des langages les plus populaires pour interagir avec PostgreSQL, notamment grâce à la bibliothèque **psycopg**, qui est le driver PostgreSQL le plus utilisé dans l'écosystème Python. Cette section vous guide à travers l'utilisation de **psycopg3**, la version moderne et recommandée, tout en mentionnant **psycopg2** pour contexte historique.

---

## Qu'est-ce que psycopg ?

**psycopg** est un adaptateur PostgreSQL pour Python qui permet à vos applications Python de communiquer avec une base de données PostgreSQL. Il implémente la spécification **DB-API 2.0** (PEP 249), qui est le standard Python pour les interfaces de bases de données.

### Historique : psycopg2 vs psycopg3

| Caractéristique | psycopg2 (Legacy) | psycopg3 (Moderne) |
|----------------|-------------------|-------------------|
| **Première version** | 2006 | 2021 |
| **Support Python** | Python 2.7 - 3.11+ | Python 3.7+ uniquement |
| **Architecture** | Synchrone uniquement | Synchrone **et** asynchrone |
| **Performance** | Bonne | Excellente (optimisations modernes) |
| **Syntaxe** | Ancienne API | API modernisée et plus pythonique |
| **Support PostgreSQL** | Jusqu'à PostgreSQL 15 | Support complet PostgreSQL 18 |
| **Maintenance** | Maintenance limitée | Développement actif |
| **Recommandation** | Migration conseillée | **Choix par défaut** pour nouveaux projets |

> **💡 Important pour débutants :** Si vous démarrez un nouveau projet aujourd'hui, utilisez **psycopg3**. Psycopg2 n'est plus recommandé sauf pour maintenir des projets existants.

## Installation de psycopg3

### Installation standard (synchrone)

Pour installer psycopg3 avec pip :

```bash
pip install psycopg
```

Cette commande installe la version pure Python de psycopg3, qui fonctionne immédiatement sans dépendances système supplémentaires.

### Installation avec binaires C (performance optimale)

Pour de meilleures performances en production, installez la version avec les extensions C compilées :

```bash
pip install "psycopg[binary]"
```

**Différence :** La version binaire utilise du code C optimisé pour les opérations critiques, offrant jusqu'à 20-30% de performance supplémentaire.

### Installation pour le support asynchrone

Si vous développez des applications asynchrones (avec `asyncio`) :

```bash
pip install "psycopg[binary,pool]"
```

L'option `pool` ajoute le support du connection pooling, essentiel pour les applications performantes.

### Vérification de l'installation

```python
import psycopg

print(f"Version de psycopg : {psycopg.__version__}")
```

## Concepts Fondamentaux

### 1. Le modèle de connexion

Avant d'exécuter des requêtes SQL, vous devez établir une **connexion** avec PostgreSQL. Cette connexion représente une session entre votre application Python et le serveur de base de données.

**Schéma conceptuel :**
```
Application Python
      ↓
[psycopg3 Driver]
      ↓
[Connexion réseau]
      ↓
Serveur PostgreSQL
```

### 2. Les curseurs (Cursors)

Un **curseur** est un objet qui permet d'exécuter des requêtes SQL et de récupérer les résultats. Pensez au curseur comme un "pointeur" qui parcourt les résultats d'une requête.

**Analogie :** Si la connexion est comme une "ligne téléphonique" avec la base de données, le curseur est comme le "microphone" que vous utilisez pour parler et écouter.

### 3. Les transactions

Par défaut, PostgreSQL fonctionne en mode transactionnel. Chaque opération (INSERT, UPDATE, DELETE) fait partie d'une transaction qui doit être **validée** (COMMIT) ou **annulée** (ROLLBACK).

## Connexion à PostgreSQL

### Syntaxe de base

```python
import psycopg

# Chaîne de connexion (Connection String)
conn = psycopg.connect(
    host="localhost",        # Adresse du serveur PostgreSQL
    port=5432,              # Port (5432 par défaut)
    dbname="ma_database",   # Nom de la base de données
    user="mon_utilisateur", # Nom d'utilisateur
    password="mon_password" # Mot de passe
)

print("Connexion établie avec succès !")

# Important : toujours fermer la connexion
conn.close()
```

### Utilisation du context manager (recommandé)

Le **context manager** (mot-clé `with`) garantit que la connexion sera automatiquement fermée, même en cas d'erreur :

```python
import psycopg

with psycopg.connect(
    host="localhost",
    dbname="ma_database",
    user="mon_utilisateur",
    password="mon_password"
) as conn:
    print("Connexion active")
    # Votre code ici
# La connexion est automatiquement fermée ici
```

**Avantage :** Vous n'avez plus besoin d'appeler `conn.close()` manuellement.

### Chaîne de connexion (Connection String)

Vous pouvez également utiliser une **chaîne de connexion** unique, appelée DSN (Data Source Name) ou connstring :

```python
import psycopg

# Format libpq (PostgreSQL standard)
connstring = "postgresql://mon_utilisateur:mon_password@localhost:5432/ma_database"

with psycopg.connect(connstring) as conn:
    print("Connecté via connstring")
```

**Format de la connstring :**
```
postgresql://[user]:[password]@[host]:[port]/[database]?[options]
```

**Exemple avec SSL :**
```python
connstring = "postgresql://user:pass@localhost/db?sslmode=require"
```

## Exécution de Requêtes SQL

### 1. Requêtes simples (SELECT)

```python
import psycopg

with psycopg.connect("postgresql://user:pass@localhost/db") as conn:
    with conn.cursor() as cur:
        # Exécuter une requête SELECT
        cur.execute("SELECT version()")

        # Récupérer le résultat (une seule ligne)
        result = cur.fetchone()
        print(f"Version PostgreSQL : {result[0]}")
```

**Explication :**
- `conn.cursor()` : Crée un curseur pour exécuter des requêtes  
- `cur.execute()` : Exécute la requête SQL  
- `cur.fetchone()` : Récupère une seule ligne de résultat (renvoie un tuple)

### 2. Récupération de plusieurs lignes

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute("SELECT id, nom, email FROM utilisateurs")

        # Récupérer toutes les lignes
        rows = cur.fetchall()

        for row in rows:
            print(f"ID: {row[0]}, Nom: {row[1]}, Email: {row[2]}")
```

**Méthodes de récupération :**
- `fetchone()` : Récupère une seule ligne (ou `None` si plus de résultats)  
- `fetchall()` : Récupère toutes les lignes restantes (retourne une liste)  
- `fetchmany(size)` : Récupère `size` lignes (utile pour traiter par batch)

### 3. Itération sur les résultats

Pour économiser la mémoire avec de grandes tables, vous pouvez itérer directement sur le curseur :

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute("SELECT * FROM grande_table")

        # Itération ligne par ligne (économe en mémoire)
        for row in cur:
            print(row)
```

**Avantage :** Les lignes sont chargées progressivement, évitant de saturer la mémoire.

### 4. Utilisation de dictionnaires (Row Factories)

Par défaut, les résultats sont des tuples. Pour avoir des dictionnaires (plus lisibles), utilisez un **row factory** :

```python
from psycopg.rows import dict_row

with psycopg.connect(connstring) as conn:
    with conn.cursor(row_factory=dict_row) as cur:
        cur.execute("SELECT id, nom, email FROM utilisateurs")

        for row in cur:
            print(f"ID: {row['id']}, Nom: {row['nom']}, Email: {row['email']}")
```

**Autres row factories disponibles :**
- `tuple_row` : Comportement par défaut (tuple)  
- `dict_row` : Dictionnaire (clé = nom de colonne)  
- `namedtuple_row` : Named tuple (accès par attribut : `row.nom`)  
- `class_row(MyClass)` : Mapper vers une classe personnalisée

## Paramétrage des Requêtes (Sécurité Critique)

### ⚠️ Le danger de l'injection SQL

**JAMAIS comme ceci (DANGEREUX) :**
```python
# ❌ CODE VULNÉRABLE - NE JAMAIS FAIRE ÇA !
nom_utilisateur = "Alice' OR '1'='1"  # Injection SQL malveillante  
cur.execute(f"SELECT * FROM users WHERE nom = '{nom_utilisateur}'")  
```

Cette approche permet des **injections SQL**, l'une des vulnérabilités les plus graves en sécurité informatique.

### ✅ La bonne méthode : Requêtes paramétrées

psycopg3 utilise le symbole `%s` comme **placeholder** pour les paramètres :

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        # ✅ Méthode sécurisée
        nom = "Alice"
        cur.execute(
            "SELECT * FROM utilisateurs WHERE nom = %s",
            (nom,)  # Tuple de paramètres
        )
        result = cur.fetchone()
```

**Important :**
- Utilisez toujours `%s` (même pour les nombres)
- Les paramètres sont passés dans un **tuple** ou une **liste**
- psycopg3 échappe automatiquement les valeurs dangereuses

### Paramètres multiples

```python
with conn.cursor() as cur:
    cur.execute(
        "SELECT * FROM produits WHERE categorie = %s AND prix > %s",
        ("Électronique", 100.0)
    )
```

### Paramètres nommés (recommandé pour la lisibilité)

```python
with conn.cursor() as cur:
    cur.execute(
        "SELECT * FROM produits WHERE categorie = %(cat)s AND prix > %(prix)s",
        {"cat": "Électronique", "prix": 100.0}
    )
```

**Avantage :** Le code est plus lisible et maintenable.

## Opérations d'Écriture (INSERT, UPDATE, DELETE)

### 1. INSERT : Insérer des données

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute(
            "INSERT INTO utilisateurs (nom, email, age) VALUES (%s, %s, %s)",
            ("Bob", "bob@example.com", 30)
        )
        # Commit automatique grâce au context manager
```

### 2. INSERT avec RETURNING (spécificité PostgreSQL)

PostgreSQL permet de récupérer des valeurs après insertion (par exemple, l'ID auto-généré) :

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute(
            "INSERT INTO utilisateurs (nom, email) VALUES (%s, %s) RETURNING id",
            ("Charlie", "charlie@example.com")
        )
        nouveau_id = cur.fetchone()[0]
        print(f"Nouvel utilisateur créé avec ID : {nouveau_id}")
```

### 3. Insertions multiples (bulk insert)

Pour insérer plusieurs lignes efficacement, utilisez `executemany()` :

```python
data = [
    ("Alice", "alice@example.com", 25),
    ("Bob", "bob@example.com", 30),
    ("Charlie", "charlie@example.com", 35)
]

with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.executemany(
            "INSERT INTO utilisateurs (nom, email, age) VALUES (%s, %s, %s)",
            data
        )
        print(f"{cur.rowcount} lignes insérées")
```

**Performance :** `executemany()` est beaucoup plus rapide que des `execute()` multiples.

### 4. UPDATE : Modifier des données

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute(
            "UPDATE utilisateurs SET email = %s WHERE nom = %s",
            ("nouvel_email@example.com", "Alice")
        )
        print(f"{cur.rowcount} ligne(s) modifiée(s)")
```

### 5. DELETE : Supprimer des données

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute(
            "DELETE FROM utilisateurs WHERE age < %s",
            (18,)
        )
        print(f"{cur.rowcount} ligne(s) supprimée(s)")
```

## Gestion des Transactions

### Comportement par défaut

Par défaut, psycopg3 démarre une transaction automatiquement. Quand vous utilisez le context manager `with conn`, la transaction est :
- **Validée (COMMIT)** si tout se passe bien  
- **Annulée (ROLLBACK)** en cas d'erreur

### Gestion manuelle des transactions

```python
import psycopg

conn = psycopg.connect(connstring)

try:
    with conn.cursor() as cur:
        # Opération 1
        cur.execute("INSERT INTO comptes (nom, solde) VALUES (%s, %s)", ("Alice", 1000))

        # Opération 2
        cur.execute("INSERT INTO comptes (nom, solde) VALUES (%s, %s)", ("Bob", 500))

    # Validation explicite
    conn.commit()
    print("Transaction validée")

except Exception as e:
    # Annulation en cas d'erreur
    conn.rollback()
    print(f"Transaction annulée : {e}")

finally:
    conn.close()
```

### Transactions avec SAVEPOINT

Les **savepoints** permettent d'annuler partiellement une transaction :

```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute("INSERT INTO logs (message) VALUES (%s)", ("Opération 1",))

        # Créer un point de sauvegarde
        conn.execute("SAVEPOINT mon_savepoint")

        try:
            cur.execute("INSERT INTO logs (message) VALUES (%s)", ("Opération 2",))
            # Opération risquée qui peut échouer
            cur.execute("INSERT INTO logs (bad_column) VALUES (%s)", ("Erreur",))
        except Exception:
            # Revenir au savepoint (annule seulement après le savepoint)
            conn.execute("ROLLBACK TO SAVEPOINT mon_savepoint")
            print("Savepoint restauré")

        # Opération 1 est toujours valide
```

### Autocommit (mode non-transactionnel)

Pour désactiver les transactions automatiques :

```python
conn = psycopg.connect(connstring, autocommit=True)

with conn.cursor() as cur:
    cur.execute("INSERT INTO logs (message) VALUES (%s)", ("Log direct",))
    # Pas besoin de commit, chaque requête est immédiatement validée
```

**Usage :** Rare, utilisé pour des opérations comme `VACUUM` ou `CREATE DATABASE` qui ne peuvent pas être dans une transaction.

from datetime import date, datetime, timedelta

import json

## Connection Pooling (Gestion des Connexions)

### Pourquoi le pooling est important

Créer une nouvelle connexion à PostgreSQL est **coûteux** :
- Établissement de la connexion TCP
- Authentification
- Allocation de ressources côté serveur

Un **pool de connexions** réutilise des connexions existantes au lieu d'en créer de nouvelles à chaque requête.

### Utilisation de ConnectionPool

```python
from psycopg_pool import ConnectionPool

# Créer un pool de connexions
pool = ConnectionPool(
    conninfo="postgresql://user:pass@localhost/db",
    min_size=2,      # Minimum 2 connexions actives
    max_size=10,     # Maximum 10 connexions
    timeout=5.0      # Timeout en secondes
)

# Utiliser une connexion du pool
with pool.connection() as conn:
    with conn.cursor() as cur:
        cur.execute("SELECT * FROM utilisateurs")
        results = cur.fetchall()

# La connexion retourne automatiquement dans le pool

# Fermer le pool à la fin de l'application
pool.close()
```

### Paramètres de configuration du pool

| Paramètre | Description | Valeur recommandée |
|-----------|-------------|-------------------|
| `min_size` | Connexions toujours actives | 2-5 |
| `max_size` | Connexions maximum autorisées | 10-20 (OLTP), 5-10 (OLAP) |
| `timeout` | Temps d'attente pour obtenir une connexion | 5-30 secondes |
| `max_idle` | Durée de vie d'une connexion inactive | 300-600 secondes |
| `max_lifetime` | Durée de vie totale d'une connexion | 3600 secondes |

### Pool asynchrone (pour asyncio)

```python
from psycopg_pool import AsyncConnectionPool

# Pool pour applications asynchrones
pool = AsyncConnectionPool(
    conninfo="postgresql://user:pass@localhost/db",
    min_size=2,
    max_size=10
)

# Utilisation avec async/await
async def query_data():
    async with pool.connection() as conn:
        async with conn.cursor() as cur:
            await cur.execute("SELECT * FROM utilisateurs")
            results = await cur.fetchall()
            return results
```

## Gestion des Erreurs

### Hiérarchie des exceptions psycopg3

```
Exception (Python standard)
    └── psycopg.Error
            ├── psycopg.DatabaseError
            │       ├── psycopg.IntegrityError (violation de contrainte)
            │       ├── psycopg.OperationalError (problème de connexion)
            │       ├── psycopg.ProgrammingError (erreur SQL)
            │       └── psycopg.DataError (problème de type de données)
            └── psycopg.InterfaceError (problème avec psycopg lui-même)
```

### Gestion basique des erreurs

```python
import psycopg

try:
    with psycopg.connect(connstring) as conn:
        with conn.cursor() as cur:
            cur.execute("INSERT INTO utilisateurs (email) VALUES (%s)",
                       ("duplicate@example.com",))

except psycopg.IntegrityError as e:
    print(f"Erreur d'intégrité (contrainte violée) : {e}")

except psycopg.OperationalError as e:
    print(f"Erreur de connexion : {e}")

except psycopg.Error as e:
    print(f"Erreur PostgreSQL générique : {e}")
```

### Accéder aux détails de l'erreur

PostgreSQL fournit des informations détaillées sur les erreurs :

```python
try:
    with conn.cursor() as cur:
        cur.execute("INSERT INTO users (id, email) VALUES (%s, %s)", (1, "test@example.com"))

except psycopg.IntegrityError as e:
    print(f"Message : {e}")
    print(f"Code SQLSTATE : {e.sqlstate}")  # Ex: '23505' pour unique violation
    print(f"Détail : {e.diag.message_detail}")
    print(f"Table : {e.diag.table_name}")
    print(f"Colonne : {e.diag.column_name}")
```

**Codes SQLSTATE courants :**
- `23505` : Violation de contrainte UNIQUE  
- `23503` : Violation de contrainte FOREIGN KEY  
- `23502` : Violation de contrainte NOT NULL  
- `42P01` : Table inexistante  
- `42703` : Colonne inexistante

## Bonnes Pratiques

### 1. Toujours utiliser des context managers

✅ **Bon :**
```python
with psycopg.connect(connstring) as conn:
    with conn.cursor() as cur:
        cur.execute("SELECT * FROM users")
```

❌ **Éviter :**
```python
conn = psycopg.connect(connstring)  
cur = conn.cursor()  
cur.execute("SELECT * FROM users")  
cur.close()  
conn.close()  
```

### 2. Toujours utiliser des requêtes paramétrées

✅ **Bon :**
```python
cur.execute("SELECT * FROM users WHERE email = %s", (email,))
```

❌ **Dangereux :**
```python
cur.execute(f"SELECT * FROM users WHERE email = '{email}'")
```

### 3. Utiliser un pool de connexions en production

Pour une application web ou API :
```python
# Au démarrage de l'application
from psycopg_pool import ConnectionPool

app_pool = ConnectionPool(connstring, min_size=5, max_size=20)

# Dans vos handlers/routes
def get_user(user_id):
    with app_pool.connection() as conn:
        with conn.cursor(row_factory=dict_row) as cur:
            cur.execute("SELECT * FROM users WHERE id = %s", (user_id,))
            return cur.fetchone()
```

### 4. Gérer les erreurs de manière appropriée

```python
try:
    with conn.cursor() as cur:
        cur.execute("INSERT INTO logs (message) VALUES (%s)", (msg,))
except psycopg.OperationalError:
    # Réessayer ou logger
    logger.error("Connexion perdue, tentative de reconnexion...")
except psycopg.IntegrityError:
    # Contrainte violée, informer l'utilisateur
    return {"error": "Cette valeur existe déjà"}
```

### 5. Utiliser des transactions explicites pour les opérations critiques

```python
with conn:  # Transaction automatique
    with conn.cursor() as cur:
        cur.execute("UPDATE comptes SET solde = solde - %s WHERE id = %s", (100, 1))
        cur.execute("UPDATE comptes SET solde = solde + %s WHERE id = %s", (100, 2))
    # Commit automatique si aucune exception
```

### 6. Fermer le pool à la fin de l'application

```python
import atexit

pool = ConnectionPool(connstring)

# Enregistrer la fermeture automatique
atexit.register(pool.close)
```

### 7. Utiliser des row factories pour plus de lisibilité

```python
from psycopg.rows import dict_row, namedtuple_row

# Dictionnaire (accès par clé)
with conn.cursor(row_factory=dict_row) as cur:
    cur.execute("SELECT id, nom FROM users")
    user = cur.fetchone()
    print(user['nom'])  # Plus lisible que user[1]

# Named tuple (accès par attribut)
with conn.cursor(row_factory=namedtuple_row) as cur:
    cur.execute("SELECT id, nom FROM users")
    user = cur.fetchone()
    print(user.nom)  # Encore plus pythonique
```

## Comparaison avec psycopg2 (Migration)

Si vous avez du code existant en psycopg2, voici les principales différences :

### Import

```python
# psycopg2
import psycopg2

# psycopg3
import psycopg
```

### Context managers

```python
# psycopg2 : connexion ne supporte pas 'with' pour autocommit
conn = psycopg2.connect(...)  
try:  
    cur = conn.cursor()
    cur.execute("...")
    conn.commit()
finally:
    conn.close()

# psycopg3 : 'with' gère automatiquement commit/rollback
with psycopg.connect(...) as conn:
    with conn.cursor() as cur:
        cur.execute("...")
```

### Placeholders

```python
# psycopg2 : %s ou %(name)s
cur.execute("SELECT * FROM users WHERE id = %s", (1,))

# psycopg3 : identique, %s ou %(name)s
cur.execute("SELECT * FROM users WHERE id = %s", (1,))
```

**Bonne nouvelle :** La syntaxe des requêtes est identique ! La migration est donc assez simple.

### Row factories

```python
# psycopg2
import psycopg2.extras  
cur = conn.cursor(cursor_factory=psycopg2.extras.RealDictCursor)  

# psycopg3
from psycopg.rows import dict_row  
cur = conn.cursor(row_factory=dict_row)  
```

## Exemple Complet : Application CRUD Simple

Voici un exemple complet d'application Python avec psycopg3 :

```python
import psycopg  
from psycopg.rows import dict_row  
from psycopg_pool import ConnectionPool  

# Configuration
DATABASE_URL = "postgresql://user:password@localhost:5432/myapp"

# Pool de connexions global
pool = ConnectionPool(DATABASE_URL, min_size=2, max_size=10)

class UserRepository:
    """Gestion des utilisateurs dans la base de données"""

    def create_user(self, nom, email, age):
        """Créer un nouvel utilisateur"""
        with pool.connection() as conn:
            with conn.cursor() as cur:
                try:
                    cur.execute(
                        "INSERT INTO utilisateurs (nom, email, age) VALUES (%s, %s, %s) RETURNING id",
                        (nom, email, age)
                    )
                    user_id = cur.fetchone()[0]
                    return {"success": True, "id": user_id}
                except psycopg.IntegrityError:
                    return {"success": False, "error": "Email déjà utilisé"}

    def get_user(self, user_id):
        """Récupérer un utilisateur par ID"""
        with pool.connection() as conn:
            with conn.cursor(row_factory=dict_row) as cur:
                cur.execute("SELECT * FROM utilisateurs WHERE id = %s", (user_id,))
                user = cur.fetchone()
                return user if user else None

    def get_all_users(self):
        """Récupérer tous les utilisateurs"""
        with pool.connection() as conn:
            with conn.cursor(row_factory=dict_row) as cur:
                cur.execute("SELECT * FROM utilisateurs ORDER BY nom")
                return cur.fetchall()

    def update_user(self, user_id, nom=None, email=None, age=None):
        """Mettre à jour un utilisateur"""
        with pool.connection() as conn:
            with conn.cursor() as cur:
                updates = []
                params = []

                if nom:
                    updates.append("nom = %s")
                    params.append(nom)
                if email:
                    updates.append("email = %s")
                    params.append(email)
                if age:
                    updates.append("age = %s")
                    params.append(age)

                if not updates:
                    return False

                params.append(user_id)
                query = f"UPDATE utilisateurs SET {', '.join(updates)} WHERE id = %s"

                cur.execute(query, params)
                return cur.rowcount > 0

    def delete_user(self, user_id):
        """Supprimer un utilisateur"""
        with pool.connection() as conn:
            with conn.cursor() as cur:
                cur.execute("DELETE FROM utilisateurs WHERE id = %s", (user_id,))
                return cur.rowcount > 0

# Utilisation
if __name__ == "__main__":
    repo = UserRepository()

    # Créer
    result = repo.create_user("Alice", "alice@example.com", 30)
    print(f"Utilisateur créé : {result}")

    # Lire
    user = repo.get_user(1)
    print(f"Utilisateur : {user}")

    # Lister
    tous = repo.get_all_users()
    print(f"Tous les utilisateurs : {tous}")

    # Modifier
    repo.update_user(1, age=31)

    # Supprimer
    repo.delete_user(1)

    # Fermer le pool à la fin
    pool.close()
```

## Support Asynchrone (asyncio)

psycopg3 offre un support natif pour la programmation asynchrone avec `asyncio` :

```python
import asyncio  
import psycopg  

async def fetch_users():
    """Exemple de requête asynchrone"""
    async with await psycopg.AsyncConnection.connect(
        "postgresql://user:pass@localhost/db"
    ) as conn:
        async with conn.cursor() as cur:
            await cur.execute("SELECT * FROM utilisateurs")
            users = await cur.fetchall()
            return users

# Exécution
async def main():
    users = await fetch_users()
    print(users)

asyncio.run(main())
```

**Avantages :**
- Idéal pour les applications web asynchrones (FastAPI, aiohttp)
- Permet de gérer des milliers de connexions simultanées
- Meilleures performances pour les I/O-bound workloads

## Résumé des Points Clés

- ✅ **psycopg3** est le driver moderne recommandé pour Python + PostgreSQL  
- ✅ Utilisez toujours des **context managers** (`with`) pour gérer connexions et curseurs  
- ✅ **Sécurité** : Toujours utiliser des requêtes paramétrées (jamais de f-strings)  
- ✅ **Performance** : Utiliser un **ConnectionPool** en production  
- ✅ **Lisibilité** : Préférer `dict_row` ou `namedtuple_row` pour les résultats  
- ✅ **Transactions** : Laisser le context manager gérer commit/rollback automatiquement  
- ✅ **Erreurs** : Capturer les exceptions psycopg spécifiques (IntegrityError, OperationalError)  
- ✅ **Asynchrone** : psycopg3 supporte nativement asyncio pour les applications modernes
