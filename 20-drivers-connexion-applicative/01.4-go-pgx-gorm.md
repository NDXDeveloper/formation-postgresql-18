# 20.1.4. Go : pgx, GORM

## Introduction

Go (Golang) est un langage de programmation moderne créé par Google, particulièrement apprécié pour la construction de services backend performants et de microservices. Pour interagir avec PostgreSQL en Go, deux approches principales existent :

1. **pgx** : Driver natif PostgreSQL bas niveau, haute performance  
2. **GORM** : ORM (Object-Relational Mapping) complet et feature-rich

Ce tutoriel couvre les deux approches en profondeur pour vous permettre de choisir celle qui convient le mieux à votre projet.

---

## Vue d'Ensemble : Driver vs ORM

### Qu'est-ce qu'un Driver ?

Un **driver** (ou adaptateur) est une bibliothèque qui permet à votre code Go de communiquer directement avec PostgreSQL. Vous écrivez du SQL brut et manipulez les résultats.

**Analogie :** C'est comme parler directement à PostgreSQL dans sa langue native (SQL).

### Qu'est-ce qu'un ORM ?

Un **ORM** (Object-Relational Mapping) est une couche d'abstraction qui traduit vos structures Go en requêtes SQL. Vous manipulez des objets Go au lieu d'écrire du SQL.

**Analogie :** C'est comme avoir un traducteur automatique entre votre code Go et PostgreSQL.

### Comparaison Rapide

| Caractéristique | pgx | GORM |
|-----------------|-----|------|
| **Type** | Driver natif | ORM complet |
| **Langage** | SQL brut | API Go |
| **Performance** | Excellente (direct) | Très bonne (overhead minimal) |
| **Courbe d'apprentissage** | Moyenne (SQL requis) | Facile |
| **Contrôle** | Total | Abstraction |
| **Flexibilité SQL** | Toutes requêtes possibles | Limité aux capacités ORM |
| **Type-safety** | Manuelle | Automatique (structs) |
| **Migrations** | Manuelles | Intégrées (AutoMigrate) |
| **Boilerplate** | Plus de code | Moins de code |
| **Popularité** | Très élevée | Très élevée |
| **Maintenance** | Active | Active |

**Recommandation :**
- **pgx** : Pour des performances maximales, SQL complexe, contrôle total, microservices légers  
- **GORM** : Pour la productivité, applications CRUD standard, prototypage rapide

## Partie 1 : pgx (Driver PostgreSQL Natif)

### Introduction à pgx

**pgx** est le driver PostgreSQL le plus performant et le plus complet pour Go. Il est :
- **Rapide** : Optimisé pour PostgreSQL spécifiquement  
- **Complet** : Support de toutes les fonctionnalités PostgreSQL  
- **Idiomatique** : Suit les conventions Go  
- **Mature** : Utilisé en production par de grandes entreprises

**pgx vs lib/pq (driver standard) :**
- pgx est **2-3× plus rapide** que lib/pq
- pgx supporte les types PostgreSQL avancés (JSONB, Arrays, hstore)
- pgx offre un pool de connexions intégré
- lib/pq est maintenant en mode maintenance

### Connexion Simple

```go
package main

import (
    "context"
    "fmt"
    "log"

    "github.com/jackc/pgx/v5"
)

func main() {
    // URL de connexion PostgreSQL
    connString := "postgres://user:password@localhost:5432/mydb"

    // Établir la connexion
    conn, err := pgx.Connect(context.Background(), connString)
    if err != nil {
        log.Fatalf("Impossible de se connecter : %v\n", err)
    }
    defer conn.Close(context.Background())

    fmt.Println("✅ Connexion établie avec succès")

    // Tester la connexion
    var version string
    err = conn.QueryRow(context.Background(), "SELECT version()").Scan(&version)
    if err != nil {
        log.Fatalf("Erreur de requête : %v\n", err)
    }

    fmt.Printf("Version PostgreSQL : %s\n", version)
}
```

### Pool de Connexions (Production)

En production, **TOUJOURS** utiliser un pool de connexions :

```go
package database

import (
    "context"
    "fmt"
    "log"

    "github.com/jackc/pgx/v5/pgxpool"
)

var Pool *pgxpool.Pool

// InitDB initialise le pool de connexions
func InitDB() error {
    connString := "postgres://user:password@localhost:5432/mydb"

    // Configuration du pool
    config, err := pgxpool.ParseConfig(connString)
    if err != nil {
        return fmt.Errorf("impossible de parser la config : %w", err)
    }

    // Paramètres du pool
    config.MaxConns = 25                      // Maximum 25 connexions
    config.MinConns = 5                       // Minimum 5 connexions actives
    config.MaxConnLifetime = 1 * time.Hour    // Durée de vie max
    config.MaxConnIdleTime = 30 * time.Minute // Timeout inactivité

    // Créer le pool
    Pool, err = pgxpool.NewWithConfig(context.Background(), config)
    if err != nil {
        return fmt.Errorf("impossible de créer le pool : %w", err)
    }

    // Vérifier la connexion
    err = Pool.Ping(context.Background())
    if err != nil {
        return fmt.Errorf("impossible de ping la DB : %w", err)
    }

    log.Println("✅ Pool de connexions initialisé")
    return nil
}

// Close ferme le pool
func Close() {
    if Pool != nil {
        Pool.Close()
    }
}
```

**Insertion multiple (batch) :**

```go
func CreateUsers(ctx context.Context, users []models.CreateUserInput) error {
    // Utiliser une transaction pour le batch
    tx, err := database.Pool.Begin(ctx)
    if err != nil {
        return err
    }
    defer tx.Rollback(ctx) // Rollback si commit non appelé

    query := `
        INSERT INTO utilisateurs (nom, email, age)
        VALUES ($1, $2, $3)
    `

    // Préparer le batch
    batch := &pgx.Batch{}
    for _, user := range users {
        batch.Queue(query, user.Nom, user.Email, user.Age)
    }

    // Exécuter le batch
    results := tx.SendBatch(ctx, batch)
    defer results.Close()

    // Commit la transaction
    return tx.Commit(ctx)
}
```

```go
// GetAllUsers récupère tous les utilisateurs
func GetAllUsers(ctx context.Context) ([]models.User, error) {
    query := `
        SELECT id, nom, email, age, created_at, updated_at
        FROM utilisateurs
        ORDER BY nom
    `

    rows, err := database.Pool.Query(ctx, query)
    if err != nil {
        return nil, fmt.Errorf("erreur de requête : %w", err)
    }
    defer rows.Close()

    var users []models.User

    for rows.Next() {
        var user models.User
        err := rows.Scan(
            &user.ID,
            &user.Nom,
            &user.Email,
            &user.Age,
            &user.CreatedAt,
            &user.UpdatedAt,
        )
        if err != nil {
            return nil, fmt.Errorf("erreur de scan : %w", err)
        }
        users = append(users, user)
    }

    // Vérifier les erreurs de rows
    if err := rows.Err(); err != nil {
        return nil, fmt.Errorf("erreur de rows : %w", err)
    }

    return users, nil
}

// SearchUsers recherche avec filtres

    for rows.Next() {
```

```go
func UpdateUser(ctx context.Context, id int, input models.UpdateUserInput) (*models.User, error) {

// UpdateUserPartial met à jour uniquement les champs fournis
func UpdateUserPartial(ctx context.Context, id int, input models.UpdateUserInput) error {
    // Construction dynamique de la requête
    query := "UPDATE utilisateurs SET "
    args := []interface{}{}
    argIndex := 1

    if input.Nom != nil {
        query += fmt.Sprintf("nom = $%d, ", argIndex)
        args = append(args, *input.Nom)
        argIndex++
    }

    if input.Email != nil {
        query += fmt.Sprintf("email = $%d, ", argIndex)
        args = append(args, *input.Email)
        argIndex++
    }

    if input.Age != nil {
        query += fmt.Sprintf("age = $%d, ", argIndex)
        args = append(args, *input.Age)
        argIndex++
    }

    // Ajouter updated_at
    query += fmt.Sprintf("updated_at = NOW() WHERE id = $%d", argIndex)
    args = append(args, id)

    commandTag, err := database.Pool.Exec(ctx, query, args...)
    if err != nil {
        return err
    }

    if commandTag.RowsAffected() == 0 {
        return fmt.Errorf("utilisateur non trouvé")
    }

    return nil
}
```

### Gestion des Transactions

```go
// TransferMoney effectue un transfert avec transaction
func TransferMoney(ctx context.Context, fromAccountID, toAccountID int, amount float64) error {
    // Commencer une transaction
    tx, err := database.Pool.Begin(ctx)
    if err != nil {
        return err
    }

    // Defer rollback (ne fait rien si commit est appelé)
    defer tx.Rollback(ctx)

    // Opération 1 : Débiter le compte source
    _, err = tx.Exec(ctx,
        "UPDATE comptes SET solde = solde - $1 WHERE id = $2",
        amount, fromAccountID)
    if err != nil {
        return fmt.Errorf("erreur de débit : %w", err)
    }

    // Opération 2 : Créditer le compte destination
    _, err = tx.Exec(ctx,
        "UPDATE comptes SET solde = solde + $1 WHERE id = $2",
        amount, toAccountID)
    if err != nil {
        return fmt.Errorf("erreur de crédit : %w", err)
    }

    // Valider la transaction
    err = tx.Commit(ctx)
    if err != nil {
        return fmt.Errorf("erreur de commit : %w", err)
    }

    return nil
}
```

**Transaction avec fonction callback :**

```go
// WithTransaction exécute une fonction dans une transaction
func WithTransaction(ctx context.Context, fn func(pgx.Tx) error) error {
    tx, err := database.Pool.Begin(ctx)
    if err != nil {
        return err
    }
    defer tx.Rollback(ctx)

    if err := fn(tx); err != nil {
        return err
    }

    return tx.Commit(ctx)
}

// Utilisation
err := WithTransaction(ctx, func(tx pgx.Tx) error {
    // Opération 1
    _, err := tx.Exec(ctx, "INSERT INTO logs (message) VALUES ($1)", "Log 1")
    if err != nil {
        return err
    }

    // Opération 2
    _, err = tx.Exec(ctx, "INSERT INTO logs (message) VALUES ($1)", "Log 2")
    if err != nil {
        return err
    }

    return nil
})
```

### Gestion des Erreurs

```go
import (
    "errors"
    "github.com/jackc/pgx/v5/pgconn"
)

func CreateUser(ctx context.Context, user models.CreateUserInput) error {
    _, err := database.Pool.Exec(ctx,
        "INSERT INTO utilisateurs (nom, email) VALUES ($1, $2)",
        user.Nom, user.Email)

    if err != nil {
        // Cast vers pgconn.PgError pour les détails
        var pgErr *pgconn.PgError
        if errors.As(err, &pgErr) {
            switch pgErr.Code {
            case "23505": // Unique violation
                return fmt.Errorf("email déjà utilisé")
            case "23503": // Foreign key violation
                return fmt.Errorf("référence invalide")
            case "23502": // NOT NULL violation
                return fmt.Errorf("champ obligatoire manquant")
            default:
                return fmt.Errorf("erreur DB: %s", pgErr.Message)
            }
        }
        return err
    }

    return nil
}
```

### Exemple Complet : API HTTP avec pgx

```go
package main

import (
    "context"
    "encoding/json"
    "log"
    "net/http"
    "strconv"

    "github.com/gorilla/mux"
    "myproject/database"
    "myproject/models"
    "myproject/repository"
)

func main() {
    // Initialiser la base de données
    if err := database.InitDB(); err != nil {
        log.Fatal(err)
    }
    defer database.Close()

    // Router
    r := mux.NewRouter()

    // Routes
    r.HandleFunc("/users", GetUsersHandler).Methods("GET")
    r.HandleFunc("/users/{id}", GetUserHandler).Methods("GET")
    r.HandleFunc("/users", CreateUserHandler).Methods("POST")
    r.HandleFunc("/users/{id}", UpdateUserHandler).Methods("PUT")
    r.HandleFunc("/users/{id}", DeleteUserHandler).Methods("DELETE")

    // Démarrer le serveur
    log.Println("🚀 Serveur démarré sur :8080")
    log.Fatal(http.ListenAndServe(":8080", r))
}

// GetUsersHandler récupère tous les utilisateurs
func GetUsersHandler(w http.ResponseWriter, r *http.Request) {
    users, err := repository.GetAllUsers(r.Context())
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(users)
}

// GetUserHandler récupère un utilisateur par ID
func GetUserHandler(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    id, err := strconv.Atoi(vars["id"])
    if err != nil {
        http.Error(w, "ID invalide", http.StatusBadRequest)
        return
    }

    user, err := repository.GetUserByID(r.Context(), id)
    if err != nil {
        http.Error(w, err.Error(), http.StatusNotFound)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(user)
}

// CreateUserHandler crée un utilisateur
func CreateUserHandler(w http.ResponseWriter, r *http.Request) {
    var input models.CreateUserInput
    if err := json.NewDecoder(r.Body).Decode(&input); err != nil {
        http.Error(w, "JSON invalide", http.StatusBadRequest)
        return
    }

    user, err := repository.CreateUser(r.Context(), input)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    json.NewEncoder(w).Encode(user)
}

func UpdateUserHandler(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    id, err := strconv.Atoi(vars["id"])
    if err != nil {
        http.Error(w, "ID invalide", http.StatusBadRequest)
        return
    }

    var input models.UpdateUserInput
    if err := json.NewDecoder(r.Body).Decode(&input); err != nil {
        http.Error(w, "JSON invalide", http.StatusBadRequest)

    user, err := repository.UpdateUser(r.Context(), id, input)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(user)
}

// DeleteUserHandler supprime un utilisateur
func DeleteUserHandler(w http.ResponseWriter, r *http.Request) {

    id, err := strconv.Atoi(vars["id"])
    if err != nil {
```

## Partie 2 : GORM (ORM pour Go)

### Introduction à GORM

**GORM** est l'ORM (Object-Relational Mapping) le plus populaire pour Go. Il offre :
- **Simplicité** : API intuitive et moins de boilerplate  
- **Productivité** : Migrations automatiques, relations, hooks  
- **Puissance** : Requêtes complexes, transactions, soft deletes  
- **Extensibilité** : Plugins et callbacks personnalisables

### Installation

```bash
# GORM core
go get -u gorm.io/gorm

# Driver PostgreSQL pour GORM
go get -u gorm.io/driver/postgres
```

### Configuration et Connexion

```go
package database

import (
    "log"

    "gorm.io/driver/postgres"
    "gorm.io/gorm"
    "gorm.io/gorm/logger"
)

var DB *gorm.DB

// InitDB initialise la connexion GORM
func InitDB() error {
    // DSN (Data Source Name)
    dsn := "host=localhost user=user password=password dbname=mydb port=5432 sslmode=disable"

    // Configuration GORM
    config := &gorm.Config{
        Logger: logger.Default.LogMode(logger.Info), // Log des requêtes
    }

    // Connexion
    var err error
    DB, err = gorm.Open(postgres.Open(dsn), config)
    if err != nil {
        return err
    }

    // Configuration du pool de connexions
    sqlDB, err := DB.DB()
    if err != nil {
        return err
    }

    sqlDB.SetMaxOpenConns(25)                      // Max 25 connexions
    sqlDB.SetMaxIdleConns(5)                       // Max 5 connexions idle
    sqlDB.SetConnMaxLifetime(time.Hour)            // Durée de vie max
    sqlDB.SetConnMaxIdleTime(30 * time.Minute)     // Timeout idle

    log.Println("✅ GORM initialisé avec succès")
    return nil
}
```

### Définition des Modèles (Models)

```go
package models

import (
    "time"
    "gorm.io/gorm"
)

// User représente un utilisateur
type User struct {
    ID        uint           `gorm:"primaryKey" json:"id"`
    Nom       string         `gorm:"size:100;not null" json:"nom"`
    Email     string         `gorm:"uniqueIndex;not null" json:"email"`
    Age       int            `json:"age"`
    CreatedAt time.Time      `json:"created_at"`
    UpdatedAt time.Time      `json:"updated_at"`
    DeletedAt gorm.DeletedAt `gorm:"index" json:"-"` // Soft delete
}

// TableName spécifie le nom de la table (optionnel)
func (User) TableName() string {
    return "utilisateurs"
}

// BeforeCreate hook avant création
func (u *User) BeforeCreate(tx *gorm.DB) error {
    // Logique avant création
    log.Printf("Création de l'utilisateur : %s", u.Email)
    return nil
}

// AfterCreate hook après création
func (u *User) AfterCreate(tx *gorm.DB) error {
    log.Printf("Utilisateur créé : ID=%d", u.ID)
    return nil
}
```

**Tags GORM courants :**

| Tag | Description | Exemple |
|-----|-------------|---------|
| `primaryKey` | Clé primaire | `gorm:"primaryKey"` |
| `autoIncrement` | Auto-incrément | `gorm:"autoIncrement"` |
| `unique` | Contrainte UNIQUE | `gorm:"unique"` |
| `uniqueIndex` | Index unique | `gorm:"uniqueIndex"` |
| `not null` | NOT NULL | `gorm:"not null"` |
| `default` | Valeur par défaut | `gorm:"default:0"` |
| `size` | Taille VARCHAR | `gorm:"size:255"` |
| `index` | Index simple | `gorm:"index"` |
| `type` | Type SQL custom | `gorm:"type:varchar(100)"` |
| `-` | Ignorer le champ | `gorm:"-"` |

```go
func SearchUsers(namePattern string, minAge, maxAge int) ([]models.User, error) {
```

#### One-to-Many (Un à Plusieurs)

```go
// User a plusieurs Posts
type User struct {
    ID    uint   `gorm:"primaryKey"`

// Ajouter un post à un utilisateur existant
```

### Transactions avec GORM

```go
// Transaction basique
func TransferMoney(fromAccountID, toAccountID uint, amount float64) error {
    return database.DB.Transaction(func(tx *gorm.DB) error {
        // Opération 1 : Débiter
        if err := tx.Model(&Account{}).Where("id = ?", fromAccountID).
            Update("balance", gorm.Expr("balance - ?", amount)).Error; err != nil {
            return err
        }

        // Opération 2 : Créditer
        if err := tx.Model(&Account{}).Where("id = ?", toAccountID).
            Update("balance", gorm.Expr("balance + ?", amount)).Error; err != nil {
            return err
        }

        // Si aucune erreur, commit automatique
        return nil
    })
}
```

**Transaction manuelle :**

```go
    return err
}
```

## Comparaison : pgx vs GORM

### Performance

**Benchmark indicatif (10000 requêtes SELECT) :**
- **pgx** : ~800ms  
- **GORM** : ~1200ms

**Différence :** GORM est ~30-50% plus lent que pgx, mais reste très performant pour la plupart des applications.

### Complexité du Code

**pgx :**
```go
// Plus verbeux mais contrôle total
rows, err := db.Query(ctx, "SELECT id, nom, email FROM users WHERE age > $1", 18)  
defer rows.Close()  

var users []User  
for rows.Next() {  
    var u User
    rows.Scan(&u.ID, &u.Nom, &u.Email)
    users = append(users, u)
}
```

**GORM :**
```go
// Concis et expressif
var users []User  
db.Where("age > ?", 18).Find(&users)  
```

### Quand Utiliser Chaque Approche

#### pgx ✅

**Choisir pgx si :**
- Besoin de **performances maximales**
- Requêtes SQL **complexes** (CTEs, window functions, optimisations avancées)
- **Microservices** légers avec peu de dépendances
- Contrôle total sur le SQL généré
- Équipe **expérimentée en SQL**
- Benchmarking et optimisation critique

**Exemples de cas d'usage :**
- API haute performance (>10k req/s)
- Services de données analytiques
- Système de trading temps réel
- Batch processing massif

#### GORM ✅

**Choisir GORM si :**
- Productivité et **développement rapide**
- Application **CRUD standard**
- Équipe **moins expérimentée en SQL**
- Besoin de **migrations automatiques**
- Relations complexes entre tables
- Prototypage et MVP

**Exemples de cas d'usage :**
- API REST standard
- Application web e-commerce
- Système de gestion interne
- SaaS standard

### Approche Hybride

Vous pouvez **combiner** les deux :

```go
// Utiliser GORM pour le CRUD standard
var users []models.User  
database.DB.Where("active = ?", true).Find(&users)  

// Utiliser pgx pour les requêtes complexes
sqlDB, _ := database.DB.DB()  
rows, _ := sqlDB.Query(`  
    WITH monthly_stats AS (
        SELECT DATE_TRUNC('month', created_at) as month,
               COUNT(*) as user_count
        FROM users
        GROUP BY month
    )
    SELECT * FROM monthly_stats
    ORDER BY month DESC
    LIMIT 12
`)
```

## Résumé des Points Clés

### pgx

- ✅ **Driver natif** PostgreSQL haute performance  
- ✅ **2-3× plus rapide** que les alternatives  
- ✅ Pool de connexions intégré (`pgxpool`)  
- ✅ Support complet des types PostgreSQL (JSONB, Arrays, UUID)  
- ✅ API **bas niveau** avec contrôle total  
- ✅ Idéal pour **performances critiques** et SQL complexe

### GORM

- ✅ **ORM complet** et feature-rich  
- ✅ **Productivité** élevée, moins de boilerplate  
- ✅ **Auto-migrations** et gestion de schéma  
- ✅ Relations (One-to-Many, Many-to-Many)  
- ✅ **Hooks** et callbacks  
- ✅ **Soft deletes** intégré  
- ✅ Idéal pour **développement rapide** et CRUD standard

### Choix Final

**Pour 70% des projets Go :** GORM (productivité, maintenabilité)  
**Pour performances critiques :** pgx (contrôle, vitesse)  
**Approche hybride :** GORM pour CRUD + pgx pour requêtes complexes
