# 21.4.2. Livres Recommandés

## Introduction

La documentation officielle de PostgreSQL est excellente, mais elle ne remplace pas un **bon livre** qui structure l'apprentissage, propose des exemples progressifs et offre la vision d'un expert. Les livres permettent d'approfondir des sujets, de comprendre le "pourquoi" derrière les pratiques, et d'acquérir une expertise solide.

Cette section présente une sélection de livres recommandés pour différents profils et niveaux, avec un focus particulier sur deux ouvrages de référence : **"PostgreSQL: Up and Running"** et **"The Art of PostgreSQL"**.

---

## Critères de Sélection d'un Bon Livre

### Ce Qui Fait un Bon Livre PostgreSQL

| Critère | Importance | Pourquoi |
|---------|------------|----------|
| **Actualité** | ⭐⭐⭐⭐⭐ | PostgreSQL évolue chaque année |
| **Exemples pratiques** | ⭐⭐⭐⭐⭐ | L'apprentissage passe par la pratique |
| **Structure progressive** | ⭐⭐⭐⭐ | Du simple au complexe |
| **Datasets fournis** | ⭐⭐⭐⭐ | Pour reproduire les exemples |
| **Expertise de l'auteur** | ⭐⭐⭐⭐ | Crédibilité et profondeur |
| **Couverture des sujets** | ⭐⭐⭐ | Selon vos besoins |

### Attention à la Version

PostgreSQL publie une nouvelle version majeure chaque année. Un livre écrit pour PostgreSQL 12 peut manquer des fonctionnalités importantes de PostgreSQL 18. Vérifiez toujours :

- La **version de PostgreSQL** couverte
- La **date de publication**
- Si des **mises à jour** sont disponibles

## Les Deux Références Incontournables

### PostgreSQL: Up and Running

#### Présentation

| Caractéristique | Détail |
|-----------------|--------|
| **Auteurs** | Regina Obe et Leo Hsu |
| **Éditeur** | O'Reilly Media |
| **Édition actuelle** | 3ème édition (2017) — 4ème édition attendue |
| **Pages** | ~300 pages |
| **Niveau** | Débutant à Intermédiaire |
| **Public cible** | Développeurs, DBAs débutants, migrants d'autres SGBD |

#### Les Auteurs

**Regina Obe** et **Leo Hsu** sont les co-fondateurs de Paragon Corporation, une société de conseil spécialisée dans les bases de données et les systèmes d'information géographique (SIG). Avec plus de 15 ans d'expérience professionnelle avec PostgreSQL, ils sont également contributeurs à PostGIS, l'extension géospatiale de PostgreSQL.

Leur expertise pratique se reflète dans l'approche du livre : pragmatique, orientée solutions, et ancrée dans des cas d'usage réels.

#### Contenu du Livre

Le livre est structuré pour une prise en main rapide :

| Partie | Sujets Couverts |
|--------|-----------------|
| **1. Les Bases** | Installation, psql, pgAdmin, premiers pas |
| **2. Administration** | Rôles, privilèges, configuration, maintenance |
| **3. SQL** | Types de données, requêtes, jointures, fonctions |
| **4. Avancé** | Full-Text Search, Foreign Data Wrappers, réplication |
| **5. Extensions** | PostGIS, hstore, et l'écosystème |

#### Points Forts

✅ **Concis et efficace** : Le livre va droit au but sans s'étendre inutilement

✅ **Orientation pratique** : Chaque concept est illustré par des exemples concrets

✅ **Couverture large** : De l'installation à la réplication, tous les aspects essentiels sont couverts

✅ **Références externes** : Les auteurs pointent vers des ressources complémentaires (articles, blogs, documentation)

✅ **Adapté aux migrants** : Idéal pour ceux qui viennent de MySQL, Oracle ou SQL Server

✅ **Tâches "impossibles ailleurs"** : Montre des fonctionnalités uniques de PostgreSQL

#### Points à Considérer

⚠️ **Version** : La 3ème édition couvre jusqu'à PostgreSQL 10 — vérifiez si une édition plus récente est disponible

⚠️ **Profondeur limitée** : C'est une introduction, pas un traité exhaustif

⚠️ **Prérequis SQL** : Le livre suppose une connaissance de base du SQL

#### Pour Qui ?

Ce livre est idéal si vous :

- Débutez avec PostgreSQL
- Migrez depuis un autre SGBD
- Voulez une vue d'ensemble rapide mais complète
- Préférez un format concis (~300 pages)
- Cherchez un livre de référence à garder à portée de main

#### Citation Représentative

> "Not only will you learn about the enterprise-class features, you'll also discover that PostgreSQL is more than just a database system — it's an impressive application platform."

### The Art of PostgreSQL

#### Présentation

| Caractéristique | Détail |
|-----------------|--------|
| **Auteur** | Dimitri Fontaine |
| **Éditeur** | Auto-édité |
| **Édition actuelle** | 2ème édition (mise à jour PostgreSQL 15) |
| **Pages** | ~438 pages |
| **Niveau** | Intermédiaire à Avancé |
| **Public cible** | Développeurs d'applications |

#### L'Auteur

**Dimitri Fontaine** est un **contributeur majeur** de PostgreSQL depuis le début des années 2000. Il est notamment l'auteur de :

- La commande `CREATE EXTENSION` (le système d'extensions de PostgreSQL)
- **pgloader** : L'outil de migration de données vers PostgreSQL  
- **pg_auto_failover** : Solution de haute disponibilité automatisée

Actuellement chez Microsoft (Azure Database for PostgreSQL), Dimitri est une figure reconnue de la communauté PostgreSQL et un conférencier régulier dans les événements du monde entier.

#### La Philosophie du Livre

Le sous-titre du livre résume sa philosophie :

> "Learn to replace thousands of lines of code with simple queries!"

L'idée centrale est que **SQL est un langage de programmation à part entière**, et que les développeurs sous-estiment souvent sa puissance. Plutôt que d'extraire des données brutes et de les traiter dans le code applicatif, on peut souvent accomplir la même tâche avec une requête SQL bien construite.

#### Contenu du Livre

Le livre est organisé en 8 parties et 51 chapitres :

| Partie | Thème | Chapitres |
|--------|-------|-----------|
| **I** | Préface et Introduction | Pourquoi SQL, pourquoi PostgreSQL |
| **II** | Écrire des Requêtes SQL | SELECT, WHERE, JOIN, sous-requêtes |
| **III** | SQL pour Développeurs | Business logic en SQL, traitement de données |
| **IV** | Modélisation des Données | Normalisation, dénormalisation, design |
| **V** | Types de Données | Arrays, JSON, hstore, types personnalisés |
| **VI** | SQL Avancé | Window functions, CTE, récursivité |
| **VII** | Extensions PostgreSQL | Full-Text Search, extensions utiles |
| **VIII** | Conclusion | Bonnes pratiques, workflow développeur |

#### Points Forts

✅ **Perspective développeur** : Écrit par un développeur pour des développeurs

✅ **Datasets réels** : 56 tables de données réelles (Open Data) pour pratiquer

✅ **334 requêtes** : Chaque exemple est une requête complète et fonctionnelle

✅ **SQL avancé** : Window functions, CTE récursives, LATERAL, etc.

✅ **Interviews d'experts** : Markus Winand, Craig Kerstiens, et d'autres

✅ **Mise à jour régulière** : L'auteur maintient le livre à jour avec les nouvelles versions

✅ **Approche philosophique** : Pas juste "comment", mais "pourquoi"

#### Points à Considérer

⚠️ **Pas pour débutants complets** : Suppose une connaissance de base de SQL

⚠️ **Auto-édité** : Quelques coquilles grammaticales signalées par certains lecteurs

⚠️ **Structure particulière** : Peut ressembler à une collection d'articles liés plutôt qu'un manuel linéaire

⚠️ **Prix variable** : Plusieurs éditions (Standard, Full, Enterprise) à différents prix

#### Les Différentes Éditions

| Édition | Contenu | Prix indicatif |
|---------|---------|----------------|
| **Standard** | PDF, ePub, MOBI + requêtes SQL | ~49€ |
| **Full** | Standard + dump PostgreSQL des datasets | ~99€ |
| **Enterprise** | Full + partage jusqu'à 15 personnes + tests | ~229€ |

#### Pour Qui ?

Ce livre est idéal si vous :

- Êtes un développeur qui veut maîtriser SQL
- Voulez réduire la complexité de votre code applicatif
- Souhaitez apprendre les fonctionnalités avancées de SQL
- Cherchez une approche "philosophique" de la base de données
- Voulez des exemples réels avec de vraies données

#### Citation Représentative

> "SQL is code. It needs to be treated as such: versioned, tested, code-reviewed, and deployed with the same rigor as your application code."

## Conseils de Lecture

### Comment Tirer le Maximum d'un Livre

1. **Pratiquez chaque exemple** : Ne vous contentez pas de lire, tapez les requêtes

2. **Utilisez les datasets fournis** : Reproduisez exactement les exemples du livre

3. **Prenez des notes** : Résumez les concepts clés dans vos propres mots

4. **Expérimentez** : Modifiez les exemples pour voir ce qui change

5. **Relisez** : Un bon livre technique mérite plusieurs lectures

### Créer un Environnement de Pratique

```bash
# Docker : Le moyen le plus simple
docker run --name pg-learning -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:18

# Connexion
psql -h localhost -U postgres
```
